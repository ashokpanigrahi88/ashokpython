Profitable demographic groups¶

Category: Clustering, Regression

Qmee is a free platform designed with both the brands it works with, and its users, in mind.

For users, Qmee is a rewarding App that allows you to share your opinions through surveys tailored to you, as well as earn instant cash rewards when shopping and searching online.
For brands, Qmee is a technology solution that gets better, faster and more accurate answers that can help inﬂuence the direction of your brand.
One of the services Qmee provides is matching users to research studies (surveys).
To do that, the company partners with multiple providers who hold surveys about many different topics and wish to ﬁnd users to give an opinion on their brands, products or alike.
At the same time, users earn a reward every time they complete a study successfully. It is expected for users to have different demographic proﬁles, and the industry assigns different reward amounts for studies depending on multiple factors, one of them being how “on demand” certain proﬁles are.

Notes¶

  • The user_id in both files match.
  • There could be multiple transactions for the same user on the same day.
  • JSON profiles need "cleaning up".

The format of the user profile is as follows:

{question_code: {“current_value”: answer_code, “last_updated_at”: updated_date_time, “previous_value": previous_value, “history": [history_values]}, …}

Where

  • question_code (numeric) is the code of the question (age, gender, etc).
  • answer_code (string) is the code or codes representing the answer for that question (I.e. Gender male). Some questions will accept multiple values (like “hobbies” for example), while others are single values by nature (like age).
  • updated_date_time (datetime) is the time of the last update for the current value.
  • previous_value (string) is the previous answer to the question if it was updated at some point (or null). If the user didn’t own a car but now it does, the previous value will be the code associated with "does not own a car".
  • history_values (string array) are the updates to current_value in the users history. The history logs the date of the change as well as the value of such change.

What is expected ?¶

The goal of this project are, but not limited to:¶

● Extract/clean the data
● Determine which data points are important enough to consider during clustering
● Find the correlation between earnings and demographic blocks, if any
● Use clustering techniques to determine average earnings for relevant groups
● Decide which groups have the best acquisition value
● Expose any other valuable information that could be used to leverage demographic groups

# Analyze Transaction File, collect stats by unique user ID 
# Get MIN , SUM, MEAN, MAX incomes by user and number of transactions 
def analyze_trans_file(p_filename: str,
                       p_sep: str=';',
                      p_returntype: str ='PROCESSED' ):
    trans_data = pd.read_csv(p_filename,p_sep)
    # get number of records in original file 
    noofrecords = len(trans_data)
    noofunique_userid = trans_data.user_id.nunique()
    noofunique_currency = trans_data.currency.nunique()
    if p_returntype == 'ORIGINAL':
        return noofrecords, noofunique_userid, noofunique_currency, trans_data
    new_trans_data =  trans_data.groupby('user_id').agg(
            min_income=("income_in_cents",'min'),
            sum_income=("income_in_cents",'sum'),
            mean_income=("income_in_cents",'mean'),
            max_income=("income_in_cents",'max'),
            noof_trans=('user_id','count')).reset_index()
    new_trans_data.mean_income = new_trans_data.mean_income.astype('int32')
    unique_mean_income = new_trans_data.mean_income.value_counts()
    print('Transctions:{}, \nUnique UserId:{}, \nUnique Currency:{}, \
              \nRecords in New Data Frame:{}'.format(
            noofrecords, noofunique_userid, noofunique_currency, len(new_trans_data)))
    return noofrecords, noofunique_userid, noofunique_currency, new_trans_data, unique_mean_income

def df_getdummies(p_data:pd.DataFrame()):

    abc = None
    noofanswers = 0

        mlb = MultiLabelBinarizer()
        expandedLabelData = mlb.fit_transform(questions[column].str.split(','))
        labelClasses = mlb.classes_
        if len(labelClasses) <= 100:
            # Create a pandas.DataFrame from our output
            ab = pd.DataFrame(expandedLabelData, columns=labelClasses)
            ab.columns = [column+'.'+x for x in ab.columns]
            columns_to_keep = [x for x in ab.columns if '.-1' not in x ]

            if ctr == 0:
                abc =  ab[columns_to_keep]
            else:
                abc[columns_to_keep] = ab[columns_to_keep]
            ctr += 1
    print('No of questions:{}, NoofTotalAnswers:{}'.format(ctr, noofanswers))
    return abc

def get_best_kmeans(p_data:pd.DataFrame(), no_of_clusters:int = 30):
    inertia = []
    for i in range(1,no_of_clusters):
        kmeans = KMeans(n_clusters=i, init='k-means++', 
                       max_iter=300, n_init=10, random_state=0)
        kmeans.fit(p_data)
        inertia.append(kmeans.inertia_)
    return inertia
